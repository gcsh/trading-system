**Structural critique:** the bot is REACTIVE (each cycle: look → think → trade). MITS is RETROSPECTIVE (each cycle: look → query memory → reason → trade → measure → write back).

The institutional desk doesn't have one genius trader — it has 50 years of trade journals, post-mortems, pattern catalogs, and cohort statistics. The trader queries that corpus before every decision. **We've built the trader. We haven't built the corpus.**

## Corpus strategy — operator pushback (2026-06-05)

Original plan assumed live observations as the corpus. Operator correctly pushed back: external + historical data exist, no reason to cold-start.

**New corpus strategy (Phase 0, before MITS.1):**

1. **Historical bar replay** — run detectors against yfinance + ThetaData historical bars (5-10 years on watchlist tickers + ETF benchmarks). Compute forward returns at 1-day / 5-day / 20-day horizons. Yields ~50,000+ observations per pattern across the watchlist on day one.

2. **External cohort priors:**
   - **TA-Lib pattern outputs** — 60+ candlestick patterns out of the box (engulfing, hammer, doji, etc.). Each is a free detector with academic backing.
   - **Quantpedia** ($500/yr) — 600+ backtested strategies with win-rates by cohort. Use as Bayesian priors.
   - **Hugging Face financial datasets** — several pre-labeled pattern datasets exist (CDL patterns, breakout/breakdown labels).
   - **Academic literature** — Conrad-Kaul, Bessembinder, Carhart momentum/reversion priors. Encode as starting confidence values.

3. **Bayesian update path:** prior (historical/external) + likelihood (live observations) → posterior. Confidence interval shrinks as live evidence accumulates.

**Result:** the bot has meaningful, statistically-grounded confidence on day one. "No corpus" never has to be an excuse.

**Tradeoffs to flag in implementation:**
- **Pattern stability over time** — Bull Flag in 2010 vs 2024 may differ due to algos/HFT. Mitigation: time-bucketed confidence ("last 5 years specifically").
- **Survivorship bias** — yfinance returns currently-listed tickers. Mostly fine for our liquid universe; flag in metadata.
- **Look-ahead leakage** — detectors must only use data AVAILABLE at the timestamp they fire. Validate by reproducing detector outputs on stale-bar slices.
- **External cohort granularity** — Quantpedia gives "Bull Flag avg X%"; our self-generated corpus gives "Bull Flag on NVDA in trending_up_low_vol morning session". Use external as PRIORS, self-generated for precision.

## Architecture — five phases (MITS.1 through MITS.5) + Phase 0

### Phase 0 — Historical Corpus Bootstrap (NEW, runs FIRST) — 1-2 weeks

**Goal:** populate the knowledge graph with statistical confidence BEFORE wiring any live detector path. Eliminates cold-start problem.

**Subtasks:**
- 0.1 Wire TA-Lib (`ta-lib` Python binding) into the data pipeline. Confirms 60+ candlestick patterns available.
- 0.2 Build `backend/bot/corpus/historical_replay.py` — for each watchlist ticker + ETF benchmark (SPY, QQQ, IWM, DIA, XLK, XLF, XLE):
  - Pull 10y daily bars + 6mo intraday bars (yfinance + ThetaData fallback)
  - Run every detector against each bar in sequence
  - Persist observations to `market_observations` table
  - Compute forward returns (5min/30min/60min/1d/5d/20d) and persist to `market_outcomes` table
- 0.3 Pull Quantpedia cohort priors (or equivalent — research alternatives: Composer.trade, Backtest-rookies, AlphaArchitect). Encode in `pattern_priors` table.
- 0.4 First aggregation pass — populate `knowledge_graph` table with {symbol, pattern, regime, time_bucket, vol_state} → posterior win rate + sample size.
- 0.5 Validation: run sanity checks — does "Bull Flag in trending_up regime" have higher WR than "Bull Flag in choppy regime"? If not, detector is broken.

**Expected output:** ~500,000 observations across 13 watchlist tickers × 7 ETFs × 15 patterns × 5-10y. Knowledge graph with thousands of populated cells, each with confidence intervals.

### MITS.1 — Pattern Detector Library — 2-3 weeks

**Goal:** build 12-15 objective pattern detectors as observation generators.

**Subtasks:**
- 1.1 Create `backend/bot/detectors/` package with `base.py` (Detector ABC: `detect(ticker, bars) → List[Observation]`)
- 1.2 Build individual detectors (one file each):
  - `bull_flag.py`, `bear_flag.py` (consolidation + breakout patterns)
  - `breakout.py`, `pullback.py`, `failed_breakout.py`, `failed_breakdown.py`
  - `bos.py` (Break of Structure), `choch.py` (Change of Character)
  - `liquidity_sweep.py`, `stop_hunt.py`
  - `vwap_reclaim.py`, `vwap_rejection.py`
  - `volume_acceptance.py`, `volume_rejection.py` (HVN/LVN)
  - `iv_expansion.py`, `iv_compression.py`
  - `gex_acceleration.py` (extend existing)
  - `wave_structure.py` (Elliott 5-wave detection — defer if too complex)
- 1.3 Schema: `Observation` dataclass with `ticker, pattern, timestamp, regime, vol_state, time_bucket, features (JSON), spot, iv_rank, gex_state`
- 1.4 Unit tests per detector (synthetic bars where pattern MUST fire / MUST NOT fire)
- 1.5 Run all detectors via Phase 0 historical replay; sanity-check distribution of pattern hits per ticker per year

**Output:** every cycle, detectors fire and write observations. Initial corpus already populated from Phase 0.

---

### MITS.2 — Observation → Outcome Linker — 1 week

**Goal:** background job that pairs every observation with its forward outcome.

**Subtasks:**
- 2.1 New table `market_outcomes`: `observation_id (FK), horizon (5min/30min/60min/1d/5d/20d), entry_price, exit_price, return_pct, was_winner (bool)`
- 2.2 Nightly cron in `bot/scheduler.py`: for every observation aged ≥1d (or whatever horizon), compute actual price move that followed using bar data.
- 2.3 Backfill against Phase 0 historical observations.
- 2.4 Idempotent — re-running doesn't duplicate.

**Output:** queryable history of "every Bull Flag on NVDA, with what happened next at each horizon."

---

### MITS.3 — Knowledge Graph + Confidence Engine — 1 week

**Goal:** aggregated, queryable confidence by multi-dimensional cohort.

**Subtasks:**
- 3.1 New table `knowledge_graph`: `(symbol, pattern, regime, vol_state, time_bucket) → {sample_size, win_rate, avg_move, avg_hold_time, confidence_lower, confidence_upper, last_updated}`
- 3.2 Bayesian shrinkage same as scorecard math: `posterior_wr = (wins + prior_weight × prior_wr) / (total + prior_weight)`. Priors from external sources (Quantpedia, academic literature).
- 3.3 Nightly recompute job — populates from `market_observations + market_outcomes`
- 3.4 API endpoint `GET /knowledge/{ticker}/{pattern}` returns the confidence cell
- 3.5 UI page: "Knowledge Graph" — searchable matrix of all observed cells with sample sizes + confidence

---

### MITS.4 — Memory-Augmented AI Brain — 3-5 days

**Goal:** every Brain reasoning step gets a memory block prepended.

**Subtasks:**
- 4.1 Extend `agent_context.py:build_agent_context()` — for each ticker + active pattern, query knowledge graph for top-N most-similar historical situations (kNN over features)
- 4.2 Format as: "Memory: 347 historical matches, 71% WR, avg +2.9% in 84 min. Most-similar 5: [4W, 1L]. Loser broke down through VWAP at 30min."
- 4.3 Inject into the Brain's reasoning context. Brain prompt updated to reference "evidence" rather than "vibes."
- 4.4 Council agents also see the memory block — each can vote with explicit reference to historical analogs.

**Output:** Brain stops reasoning from first principles. It reasons OVER EVIDENCE.

---

### MITS.5 — Thesis-Health Exit Monitor — 1 week

**Goal:** replace mechanical exit logic with knowledge-driven exit logic.

**Subtasks:**
- 5.1 At trade entry, query knowledge graph for "what historical winners of this pattern looked like during the hold":
  - Did winners stay above VWAP? At what hold time did the avg-winner hit peak?
  - Did losers consistently break a structural level at minute X?
- 5.2 New `thesis_health` calculator: every cycle, check current trade trajectory vs. historical winner trajectory.
- 5.3 Score: `% of winner-conditions still met`. 100% = thesis intact, 50% = degrading, <30% = exit.
- 5.4 Wire as 7th vote in council (ThesisHealthAgent). Engine consults score on exit decisions.
- 5.5 EXIT.1's mechanical trailing stays as the SAFETY NET. Thesis health is the PRIMARY exit. Brain still overrides anytime.

**Output:** exits fire on thesis degradation ("this trade no longer looks like historical winners"), not arbitrary % moves. Premium-cap problem eliminated — winners are held as long as they keep matching the winner trajectory.

## Operating principles (carried forward from MITS vision)

1. **No theory is universally correct** — all are hypotheses to be measured.
2. **Continuous improvement** — every closed trade and every new observation updates the knowledge graph.
3. **Evidence over intuition** — Brain reasons from corpus, not from priors alone.
4. **Adaptive risk** — position size scales with evidence quality (samples × confidence), not conviction alone.
5. **No upper ceilings on winners** — EXIT.1 already implements this; MITS.5 strengthens it via thesis-health.
6. **Calibration discipline** — never claim confidence not backed by samples; Bayesian shrinkage on every cell.

## Critical addition (operator caught this) — DYNAMIC TICKER PIPELINE

The system must be reactive, not static. When the operator adds a new ticker to the watchlist (today, tomorrow, anytime), the data pipeline must AUTOMATICALLY:

1. **Detect the new ticker** via watchlist add event (existing `watchlist.add_item` route)
2. **Trigger historical corpus bootstrap** for that ticker in background (async — POST returns immediately)
   - Pull 10y daily bars + 6mo intraday bars
   - Run every detector against the history
   - Persist observations + compute forward returns
   - Aggregate into knowledge graph cells for that ticker
3. **Update the cohort matrix** so the AI Brain immediately sees the new ticker's evidence
4. **Surface "corpus ready" status** in the UI (per-ticker: building / ready / insufficient samples)
5. **Engine starts evaluating the ticker in next cycle** with knowledge-graph confidence in the agent context

This MUST extend to every downstream system:
- AI Brain context (`build_agent_context`) — includes new ticker's evidence automatically
- IV history backfill (already exists — extend to also trigger corpus bootstrap)
- Cohort matrix recomputation
- Watchlist UI shows corpus status per ticker

The watchlist becomes the **trigger** for the entire pipeline. No manual steps. No "wait a week for the corpus to fill."
